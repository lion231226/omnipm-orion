# OmniPM 密钥管理策略 (F4, v2.7.0)

## 原则

1. **零硬编码秘密**：所有 API key/token 通过环境变量注入，不在源码中出现
2. **最小权限**：每个子代理仅获得其任务所需的最小环境变量集合
3. **日志脱敏**：所有诊断日志自动过滤敏感模式（token/key/secret/credential）

## 引擎自身凭证

| 凭证 | 环境变量 | 用途 | 轮换周期 |
|------|----------|------|----------|
| Pi 模型 API Key | `PI_API_KEY` | 子代理模型调用 | 90天 |
| Exa 搜索 API Key | `EXA_API_KEY` | CDL 语义搜索 | 90天 |
| GitHub CLI Token | `GITHUB_TOKEN` | CDL GitHub搜索 | 按需 |

## 运行时安全

- 子代理进程通过 `--no-session` 隔离，不继承父进程的完整环境
- 临时文件使用 `mkdtemp` + `0o600` 权限
- 诊断日志写入 `.pi/diagnostics/` 目录（不在 Git 中）

## 审计

- 所有文件操作发射结构化事件（见 `runtime/events.ts`）
- 密钥泄露检测：启动时扫描 `.pi/` 目录 + 临时目录
