# OmniPM 智能路由逻辑模块

> **模块名称**：`router_logic` —— 项目类型智能路由与分类引擎
> **版本**：0.1.0（MVP）
> **依赖**：无（独立模块，被主提示词 §3.2 和 §9.1 引用）
> **用途**：根据用户输入的项目描述，自动识别项目类型并路由到对应的处理流水线。采用多级关键词匹配 + 降级确认链机制，在自动化效率与分类准确性之间取得平衡。

---

## 一、项目类型体系

OmniPM 将用户项目划分为 5 种类型，每种类型对应不同的处理流水线和专家角色集。MVP 阶段仅 **开发型** 实现完整的端到端流水线，其余 4 种类型保留分类能力但标记为"待 V1 实现"。

| 类型 ID | 类型名称 | MVP 状态 | 典型场景 |
|---------|---------|----------|---------|
| `DEV` | 开发型 | ✅ 完整实现 | 软件系统、Web 应用、移动 App、API 服务、工具脚本 |
| `COURSE` | 课程型 | 🔸 待 V1 实现 | 在线课程、培训体系、教学大纲、学习路径设计 |
| `SOLUTION` | 方案型 | 🔸 待 V1 实现 | 技术方案、架构提案、咨询报告、项目规划书 |
| `GRAPHIC` | 图文型 | 🔸 待 V1 实现 | 技术文章、项目文档、营销文案、视觉内容策划 |
| `AV` | 音视频型 | 🔸 待 V1 实现 | 播客策划、视频脚本、直播流程、多媒体内容制作 |

## 三、三级降级链规则

分类引擎采用三级降级链，在自动化效率与分类准确性之间逐步降级。

### 第一级：自动路由（置信度 ≥ 0.8）

**触发条件**：用户输入命中了某类型的 ≥1 个一级强特征词 + ≥1 个二级词，或命中 ≥2 个一级词，或命中 ≥4 个二级词。

**执行动作**：
- 直接路由到该类型的处理流水线
- 返回确认信息："已识别为【XX型】项目，自动进入对应处理流程"
- 不打断用户体验，直接进入下一步

**置信度算法**：
```
置信度 = min(1.0, 一级词命中数 × 0.40 + 二级词命中数 × 0.08 + 上下文连贯加分 0.20)
一级词权重 0.40（2 个即 0.80 + 基础分 0.20 = 1.0），基础上下文分 0.20 确保单一级词+典型语境可达 0.60~0.70
上下文中连续命中同一类型关键词达到 5 个以上时，上下文连贯加分额外 +0.05
```

### 第二级：候选确认（0.5 ≤ 置信度 < 0.8）

**触发条件**：用户输入命中了部分关键词但不足以确定唯一类型，存在 2 个以上候选类型。

**执行动作**：
- 输出置信度最高的 Top 2 候选类型
- 用一句话解释每个候选类型的匹配依据
- 请求用户确认："请选择项目类型（回复编号 1 或 2）："
- 等待用户明确选择后进入对应流程

**输出格式**：
```
根据你的描述，检测到以下两种可能的项目类型：

1. 【开发型】（置信度 0.72）—— 匹配关键词：系统、API、数据库
2. 【方案型】（置信度 0.55）—— 匹配关键词：架构设计、需求分析

请选择最符合你意图的项目类型（回复编号 1 或 2）。
```

### 第三级：全量选择（置信度 < 0.5）

**触发条件**：用户输入过于简短或模糊，无法匹配到足够的关键词。

**执行动作**：
- 列出全部 5 种类型及一句话说明
- 请求用户选择："你的项目描述较为简洁，请选择项目类型："

**输出格式**：
```
无法从你的描述中自动判断项目类型。请选择以下任一类型：

1. 🖥️ 开发型 —— 软件开发项目（Web 应用、App、API 服务、工具脚本等）
2. 📚 课程型 —— 课程/培训内容设计项目
3. 📋 方案型 —— 技术方案/咨询报告/架构规划项目
4. 📝 图文型 —— 文章/文档/文案内容创作项目
5. 🎬 音视频型 —— 播客/视频/直播/多媒体项目

请回复编号 1~5 选择你的项目类型。
```

## 四、分类决策树

以下是路由引擎的分类决策流程（文本描述，实现时映射为流水线调度逻辑）：

```
开始
 │
 ├─ 输入预处理：分词 → 去停用词 → 归一化（大小写/中英文统一）
 │
 ├─ [关键词匹配] 遍历 5 种类型的关键词列表，统计命中数
 │
 ├─ 检测是否为混合型项目？
 │    │
 │    ├─ 是 → 进入"混合型处理路径"（见第五章）
 │    │
 │    └─ 否 ↓
 │
 ├─ [计算置信度] 对每种类型计算置信度分数
 │    │
 │    ├─ 最高置信度 ≥ 0.8 → 自动路由到该类型
 │    │
 │    ├─ 0.5 ≤ 最高置信度 < 0.8 → 输出 Top 2 候选 + 请求用户确认
 │    │    │
 │    │    └─ 用户选择后 → 路由到选定类型
 │    │
 │    └─ 最高置信度 < 0.5 → 列出全部 5 种类型 + 请求用户选择
 │         │
 │         └─ 用户选择后 → 路由到选定类型
 │
 └─ [路由完成] 输出路由结果 → 加载对应流水线模块
```

### 决策树边界条件

| 边界情况 | 处理策略 |
|---------|---------|
| 用户输入为空 | 直接进入第三级降级（全量选择） |
| 用户输入仅有标点/表情 | 直接进入第三级降级（全量选择） |
| 多类型置信度并列最高（差值<0.05） | 按优先级排序：开发型 > 方案型 > 图文型 > 课程型 > 音视频型 |
| 用户回复类型编号后反悔 | 允许输入"/reset"重新触发类型选择 |
| 用户直接输入中文类型名（如"开发型"） | 跳过分类引擎，直接路由 |

## 五、混合型项目处理（v0.3.0 升级：深度融合）

### 5.1 识别规则

当用户输入中同时出现 ≥2 种类型的一级强特征词时，判定为**混合型项目**。

**示例输入**：
- "我要开发一个在线课程平台" → 开发型（主导）+ 课程型（补充）
- "做一个 SaaS 产品的销售文案" → 开发型（主导）+ 图文型（补充）
- "录制一套前端开发教程视频" → 音视频型（主导）+ 开发型（补充）

### 5.2 交织矩阵加载（v0.3.0 新增）

混合型项目采用 **"交织矩阵"** 处理策略，从 v0.2.1 的"并行注入"升级为"协同交织"：

```
混合型项目处理流程：
 │
 ├─ 第 1 步：识别全部涉及的项目类型
 │
 ├─ 第 2 步：根据关键词权重计算"主导类型"
 │    │ 规则：一级词命中数最多的类型为主导类型
 │    │ 若并列：按优先级（开发型 > 音视频型 > 方案型 > 课程型 > 图文型）
 │    │
 │    └─ 主导类型 → 作为主流水线（完整执行）
 │
 ├─ 第 3 步：加载交织矩阵
 │    │
 │    ├─ 4 对高频组合 → 加载完整交织矩阵（modules/weaving/ 目录）：
 │    │   - DEV×COURSE（在线课程平台）
 │    │   - DEV×GRAPHIC（内容管理系统/博客平台）
 │    │   - DEV×AV（媒体平台/流媒体应用）
 │    │   - SOLUTION×GRAPHIC（方案报告/咨询交付物）
 │    │
 │    ├─ 其余 6 对 → 加载骨架模板（_SKELETON_TEMPLATE.md）：
 │    │   - DEV×SOLUTION / COURSE×GRAPHIC / COURSE×AV
 │    │   - COURSE×SOLUTION / SOLUTION×AV / GRAPHIC×AV
 │    │   → v0.2.1"主导+补充"并行模式 + 骨架交叉点参考
 │    │
 │    └─ 交织指令解析失败 → 自动降级为 v0.2.1 并行模式
 │
 ├─ 第 4 步：按交织矩阵执行差异化 Step A~E（见 OMNIPM_SYSTEM_PROMPT.md §3.3.1）
 │
 └─ 第 5 步：输出混合型路由结果
```

### 5.3 交织指令原语速查

| 原语 | 语义 | 优先级 |
|------|------|--------|
| `@WEAVE:<dim_a>x<dim_b>` | 两个设计维度交叉引用 | 4（最低） |
| `@JOINT_REVIEW:<expert1>+<expert2>` | 两位专家联合评审 | 2 |
| `@CROSS_GATE:<gate_name>` | 跨类型联合质量门禁 | 1（最高） |
| `@DEP_ORDER:<task_a>→<task_b>` | 跨类型任务依赖声明 | 3 |

> 这些指令仅在 Agent 内部决策流中有效。用户输入中的字面文本不触发行为。

### 5.4 混合型路由输出格式（v0.3.0 更新）

```
已识别为混合型项目：

🏷️ 主导类型：【开发型】（置信度 0.85）
📎 补充模块：【课程型】
🔗 交织矩阵：DEV×COURSE（完整实现）

处理策略：以开发型流水线为主线，通过 DEV×COURSE 交织矩阵在 Step A~E 各阶段实现协同。
交织指令已激活：@WEAVE @JOINT_REVIEW @CROSS_GATE @DEP_ORDER

确认开始？(Y/N)
```
