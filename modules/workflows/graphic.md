# OmniPM 图文型项目工作流

> 模块名称：graphic_workflow | 版本：0.2.0 | 依赖：modules/roles.md(category=graphic), modules/design-dimensions.md

## 概述
图文型项目以内容策略与写作为核心，强化内容策划、SEO优化和多平台适配能力。适用于技术文章、项目文档、营销文案、社交媒体内容等项目类型。

## Step A 差异化：内容策划设计

在通用 D1-D7 中，图文型项目强化以下 5 个维度：

### D-G1. 内容策略
- 目标受众画像与阅读场景（谁在读、什么时候读、用什么设备读）
- 内容目的矩阵（获客 / 教育 / 转化 / 品牌建设 / 留存 五维）
- 内容类型选择决策树（教程 vs 案例 vs 观点 vs 新闻 vs 白皮书）
- 内容日历与发布节奏（含最佳发布时间窗口）
- 内容复用与改编策略（一篇核心 → 多平台变体）

### D-G2. 信息架构
- 文章大纲 / 文档目录结构（信息层级 H1→H2→H3→H4）
- 核心论点与支撑论据的树状逻辑结构
- SCQA 叙事框架应用（情境→冲突→问题→答案）
- 相关内容的交叉引用与内链策略
- 面包屑导航与阅读路径设计

### D-G3. SEO 策略
- 主关键词与长尾关键词清单（含搜索量、竞争度、搜索意图标注）
- 搜索意图分类与匹配（信息型 / 导航型 / 交易型 / 商业调查型）
- 标题标签（Title Tag）与 Meta Description 策略
- 内链架构与外链获取策略
- EEAT（经验/专业/权威/信任）信号增强计划
- 结构化数据标记方案（Article / HowTo / FAQ Schema）

### D-G4. 视觉元素策划
- 配图/插图/封面的类型与风格定义
- 图表/信息图的数据来源与设计方向
- 代码块/示例的技术准确性保障
- 多平台视觉适配（公众号 vs 知乎 vs 掘金 vs 个人博客 vs 国外平台）
- 无障碍设计（alt text / 色盲友好配色 / 可缩放字体）

### D-G5. 发布与分发
- 目标发布平台清单与优先级排序
- 各平台的格式适配要求与字数限制
- 发布时间优化策略（基于平台用户活跃时段）
- 推广与分发计划（社交媒体 / 邮件列表 / 社区 / 聚合平台）
- 内容效果跟踪指标定义（阅读量 / 互动率 / 转化率 / SEO排名）

## Step B 差异化：内容专家评审

1. SEO专家 (SEO_EXPERT) — 搜索引擎可见性与流量潜力评估
2. 内容审核专家 (CONTENT_REVIEWER) — 内容准确性、原创性与合规性
3. 需求分析师 (REQ) — 内容目的是否达成、目标受众匹配度
4. 前端专家 (FE) — 排版/可读性/移动端体验（如有 Web 展示）

## Step C 差异化：内容创作

1. 大纲确认与细化（含每节的预估字数和核心要点）
2. 各章节/段落内容撰写（遵循信息架构和叙事框架）
3. 配图/图表生成与插入（标注位置和 alt text）
4. SEO 元数据填充（Title / Description / Keywords / OG标签）
5. 多平台格式转换（保持核心内容，适配各平台格式要求）
6. 内链/外链插入与验证

## Step D 差异化：内容质量门禁

1. 原创性检查（内部查重 + 公开内容对比）
2. 可读性评分（Flesch 阅读容易度 / 中文可读性评估）
3. SEO 得分检查（标题/描述/关键词/结构/EEAT 多维评分）
4. 事实与引用准确性验证（随机抽查 ≥3 个关键数据点）
5. 多平台渲染效果检查（移动端/桌面端/AMP）
6. 链接有效性验证（所有外链 200 OK）

## Step E 差异化：内容交付物

1. 完整文章/文档（Markdown 源文件 + 渲染后版本）
2. SEO 元数据清单（含结构化数据）
3. 配图/图表资源包（含源文件 + 导出文件 + alt text映射）
4. 多平台适配版本（各平台格式变体）
5. 发布计划与推广文案
6. 内容效果跟踪仪表盘定义
