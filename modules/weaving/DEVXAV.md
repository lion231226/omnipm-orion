# 混合型交织矩阵：DEV × AV（开发型主导 + 音视频型补充）

> 典型场景：视频课程平台、播客托管平台、直播系统、媒体资产管理系统
> 版本：0.3.0 | 状态：完整实现

---

## 交织概览

```
主导流水线：DEV（开发型 — 媒体平台/流媒体应用）
补充模块：AV（音视频型 — 内容策划与制作）
交织策略：媒体技术架构 × 内容制作流程 → 存储/转码/分发 + 脚本/分镜/后期协同
```

## Step A 交织点

| DEV 维度 | AV 维度 | 交织指令 | 说明 |
|----------|--------|---------|------|
| D1 数据架构 | D-AV3 技术方案 | `@WEAVE:data_x_media_tech` | 媒体文件存储（对象存储/CDN/转码流水线）的数据模型和元数据结构 |
| D4 性能 | D-AV2 脚本叙事 | `@WEAVE:perf_x_narrative` | 流媒体自适应码率（ABR）的性能预算 + 内容章节标记的时间轴对齐 |
| D6 部署 | D-AV4 后期流程 | `@WEAVE:deploy_x_postprod` | 转码/字幕/缩略图 自动化流水线的 CI/CD 集成 |

## Step B 联合评审会

| 联合议题 | DEV 专家 | AV 专家 | 输出 |
|---------|---------|--------|------|
| 视频播放器架构 | 前端专家 (FE) | 媒体制作专家 (MEDIA_PRODUCER) | 播放器功能需求（章节/字幕/速度/画质切换）+ 前端性能 |
| 媒体处理流水线 | 后端专家 (BE) | 媒体制作专家 (MEDIA_PRODUCER) | 上传→转码→字幕生成→分发 的技术选型和异步任务设计 |
| 内容分发策略 | DevOps (OPS) | SEO专家 (SEO_EXPERT) | CDN选型 + 多平台分发 + 媒体SEO（视频sitemap、缩略图优化） |

## Step C 依赖排序

```
DEP_ORDER:
  对象存储集成 → 媒体上传组件
  转码流水线 → 播放器组件
  字幕生成API → 字幕渲染器
  用户系统 → 播放进度追踪
  内容审核流程 → 内容发布
```

## Step D 联合门禁

```yaml
joint_gates:
  - id: "playback_quality"
    description: "播放体验 + 制作质量标准"
    checks:
      - dev: "首帧时间 < 1s, 缓冲率 < 0.5%, 码率自适应切换无黑屏"
      - av: "音频响度标准化（-16 LUFS）, 字幕同步偏移 < 200ms"
  - id: "media_accessibility"
    description: "无障碍 + 技术可靠性"
    checks:
      - dev: "键盘可操作播放器, 屏幕阅读器兼容, 字幕/画外音切换"
      - av: "字幕覆盖率 100%, 关键视觉信息有音频描述"
  - id: "storage_and_delivery"
    description: "存储成本 + 分发效率"
    checks:
      - dev: "存储分层（热/冷）, CDN缓存命中率 > 90%"
      - av: "多清晰度档位完整（1080p/720p/480p/音频-only）"
```
